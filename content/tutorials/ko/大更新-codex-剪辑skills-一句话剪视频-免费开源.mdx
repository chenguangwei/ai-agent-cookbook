# 대규모 업데이트! Codex × 편집 Skills, 한 마디로 동영상 편집, 무료 오픈소스

저는 이전에 2000개 이상의 Star를 받은 편집 Skills를 만든 적이 있습니다.

![](https://pbs.twimg.com/media/HLZPxwmW8AAM-ku.jpg)

이번에 Codex와 연결하여 대규모 업그레이드를 진행했습니다. 이제 후속 작업을 이어받아 스토리보드, 애니메이션 및 합성을 수행할 수 있습니다.

이미 여러 개의 천 개 좋아요 동영상을 만들어냈습니다.

![](https://pbs.twimg.com/media/HLZQQcEWoAA7HuX.jpg)

예전에는 동영상을 만들려면 먼저 CapCut(剪映)을 열어야 했습니다.

이제 Skills를 설치하고, 동영상과 대본을 Agent에 넘기면 전체 클립을 자동으로 처리합니다.

## 어떻게 했을까요?

첫 번째 단계는 Agent가 편집 Skills를 설치하도록 하는 것입니다.

프롬프트는 매우 간단합니다:

> chengfeng-videocut-skills 편집 Skills를 설치하거나 업데이트해 주세요. 설치 주소는 https://github.com/Agentchengfeng/chengfeng-videocut-skills 입니다.

Agent가 먼저 확인을 요청한 후, 직접 설치를 진행합니다. 설치가 완료되면 로컬에서 이 편집 Skills를 호출할 수 있습니다.

![](https://pbs.twimg.com/media/HLZQmbpW8AAZlGg.jpg)

## 편집된 동영상 및 자막 생성

두 번째 단계는 원본 음성 녹화를 기본素材 패키지로 정리하는 것입니다.

![](https://pbs.twimg.com/media/HLZRMaKXAAA4H3I.jpg)

음성 녹화 동영상과 대본을 "剪口播" Skills에 전달하면, 편집된 동영상과 자막 파일을 얻을 수 있습니다.

음성 녹화 동영상은 복잡하게 녹화할 필요가 없습니다. 저는 보통 대본을 보고 한 번 읽습니다. 중간에 특정 작업이 필요한 경우 화면을 전환하여 작업 과정을 녹화합니다.

프로젝트에서 슬래시 명령어를 입력합니다:

> /剪口播 + 동영상 주소 + 대본 주소

명령을 받으면 Agent가 먼저 검토 페이지를 생성합니다.

![](https://pbs.twimg.com/media/HLZRfCxXkAAZeoc.jpg)

Agent가 멈춤, 말실수, 다시 말하기 등을 먼저 정리합니다. 제가 해야 할 일은 이러한 삭제 항목이 올바른지 확인하는 것입니다.

문제가 없다고 확인되면 "실행 편집"을 클릭합니다.

이 단계가 완료되면 Agent가 편집된 동영상과 자막 파일을 직접 출력합니다. 얻은 자막과 편집된 동영상은 시간적으로 정렬됩니다.

## 자막 기반 스토리보드 페이지 생성

素材 패키지가 준비되면 프로젝트에서 슬래시 명령어를 입력하여 "口播成片" Skills를 호출합니다.

> /口播成片

Agent가 HTML 스토리보드 검토 페이지를 생성합니다.

![](https://pbs.twimg.com/media/HLZRwTgXsAEceUg.jpg)

왼쪽에는 Agent가 생성한 화면이, 오른쪽에는 자막, 화면 작업,素材 출처 및 카메라 동작이 표시됩니다.

이 Skills는 스토리보드 페이지에서 실제로 세 가지 작업을 수행합니다:

![](https://pbs.twimg.com/media/HLZSG50WsAAsiO3.jpg)

먼저 자막 타임라인을 기준으로 구간을 나누고, 각 구간의 내용에 따라 화면 출처를 선택합니다.

각 문장에 대해 원본 동영상을 유지할지, 스크린샷, 제품 페이지, 결과 페이지로 바꿀지, 또는 HTML 애니메이션을 만들지 Agent가 여기서 한 번에 처리합니다.

특정 구간이 마음에 들지 않으면 Codex에 몇 번째 구간이 어디가 잘못되었는지 직접 알려주면 됩니다. 수정도 매우 빠릅니다.

예를 들어:

> 05 구간 애니메이션을 수정해 주세요. 화살표가 제목을 가리키고, 오른쪽 결과를 원으로 표시해 주세요.

Codex의 Computer Use는 페이지를 열어 왼쪽 화면과 오른쪽 음성 녹화를 확인한 후, 다시 돌아가 HTML 화면이나 주석을 수정할 수 있습니다.

제 작업은 "직접 스토리보드를 구성하는 것"에서 "스토리보드를 보고 수정 사항을 한 번에 요청하는 것"으로 바뀌었습니다.

## 타임라인 미리보기 확인

스토리보드 페이지가 확인되면 타임라인 미리보기로 넘어갑니다.

![](https://pbs.twimg.com/media/HLZSYYBW8AAsEbL.jpg)

왼쪽에는 동영상 미리보기가, 하단에는 진행률 표시줄과 텍스트 설명이, 오른쪽에는 음성 녹화 내용이 표시됩니다.

타임라인 미리보기는 자막 시간 지점에 따라 애니메이션을 배치합니다. 자막이 특정 부분을 말하면 해당 애니메이션이 나타나므로, 편집 소프트웨어에서 직접 타임라인을 맞추는 것보다 훨씬 효율적입니다.

여기서 확인하는 것은: 이러한 화면들이 전체 동영상에 배치되었을 때 올바르게 나타나는지 여부입니다.

동영상에 오류가 있으면 긴 글을 작성할 필요 없이 구간별로 간단히 말하면 됩니다:

> 01 원본 동영상 유지. 02 그림이 너무 일찍 나타남. 03 화면이 너무 복잡하니 아래 두 줄 텍스트 삭제.

이 단계가 확인된 후에야 최종 합성으로 넘어갑니다.

## HyperFrames를 사용하여 MP4 합성

이전 동영상 미리보기가 확인되면 Agent가 합성을 진행하도록 할 수 있습니다.

Agent는 HyperFrames를 사용하여 이를 렌더링 가능한 동영상 애니메이션 프로젝트로 변환합니다.

![](https://pbs.twimg.com/media/HLZSp49XYAAAkFW.jpg)

HyperFrames는 Agent에 매우 친화적입니다. HTML 애니메이션을 동영상으로 변환할 수 있기 때문입니다. 화면이 HTML로 만들 수 있는 것이라면 이 합성 파이프라인에 포함될 수 있습니다.

몇 분 후 최종 애니메이션을 볼 수 있습니다.

## 편집 Agent, 전통적인 편집을 점차 대체

예전에는 동영상 제작이 타임라인을 중심으로 이루어졌습니다.

이제 동영상 제작은 워크플로우를 중심으로 이루어지기 시작했습니다.

편집 Agent는 전통적인 편집의 작업 계층을 대체하여 동영상 제작을 지속적으로 재사용 가능한 자동화 프로세스로 변화시키고 있습니다.
