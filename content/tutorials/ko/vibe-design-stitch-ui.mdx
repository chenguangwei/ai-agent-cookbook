# 바이브 디자인(Vibe Design)이란? Stitch로 한 문장으로 완전한 UI를 생성하는 가이드

# 열성적인 Google 팬으로서 저는 Google 제품, 특히 Stitch와 NotebookLM을 매우 좋아합니다.

그래서 이번에 Stitch가 큰 인기를 얻는 모습을 보니 매우 기쁩니다.

여기서 Stitch 제품이 무엇이고, 어떻게 더 잘 사용할 수 있는지 소개하겠습니다.

간단히 말해, Stitch는 "말하면 그려주는" 도구입니다. 원하는 인터페이스를 말하면 바로 디자인 목업을 생성해 주며, Figma나 AI Studio 같은 도구로 내보내기도 지원합니다.

![](https://pbs.twimg.com/media/HDwNPeqbsAA034c.jpg)

## 세 가지 주요 업데이트 하이라이트

이번 업데이트에서 제가 특히 마음에 드는 세 가지 포인트가 있습니다. UI 전체를 재설계했을 뿐만 아니라 무한 캔버스 지원도 추가했습니다.

1.  음성 상호작용

이제 마이크를 켜고 아이디어를 직접 말하기만 하면 됩니다. Stitch가 한 번에 일련의 디자인 이미지를 생성합니다.

![](https://pbs.twimg.com/media/HDwNM4pbAAELOau.jpg)

2.  라이브 미리보기

생성된 UI는 더 이상 정적 이미지가 아닙니다. 이 플레이어 UI를 예로 들면, 버튼을 직접 클릭하고 상호작용을 확인하며 실제로 먼저 경험해볼 수 있습니다.

![](https://pbs.twimg.com/media/HDwNaqqa4AAQ7Xf.jpg)

3.  통합 및 재사용 가능한 스타일

Stitch가 기존 웹사이트의 색상 구성을 읽고 추출하여 자동으로 디자인 목업을 생성하도록 할 수 있습니다. 또한 `DESIGN.md` 파일로 내보낼 수도 있습니다.

![](https://pbs.twimg.com/media/HDwNzqubwAAA1Of.jpg)

자체 프로젝트에 이미 `DESIGN.md` 파일이 있다면, Stitch에 직접 가져와 스타일 일관성을 유지할 수 있습니다.

![](https://pbs.twimg.com/media/HDwN2NtboAA_GlQ.jpg)

Google은 이에 특별히 **Vibe Design**이라는 이름을 붙였습니다. 이제 단일 프롬프트로 여러 페이지와 테마를 포함한 전체 앱 흐름을 생성할 수도 있습니다. 예를 들어, 다음 프롬프트를 보세요:

> 홈, 상품 상세, 장바구니, 결제 화면이 있는 모바일 전자상거래 앱을 만들어 줘

![](https://pbs.twimg.com/media/HDwOVHIbEAAq8TP.jpg)

제가 이전에 생성한 꽤 괜찮은 결과물입니다:

![](https://pbs.twimg.com/media/HDwOS4JboAAl-mc.jpg)

## Stitch SDK

여기서 Stitch SDK를 강력히 추천합니다. Stitch의 CLI 버전이라고 생각하면 되며, 에이전트가 호출하도록 특별히 설계되었습니다. 웹 인터페이스에서 수동으로 대화할 필요 없이, 에이전트가 프로젝트 요구 사항에 따라 자동으로 Stitch를 호출하도록 하면 됩니다. 여기서 API 키를 생성하고 일일 남은 할당량도 확인할 수 있습니다.

![](https://pbs.twimg.com/media/HDwOYVHbQAARbHj.jpg)

저는 최근 며칠 동안 이 SDK를 기반으로 래퍼 에이전트를 작업해 왔습니다:
-   백엔드 API 문서만 제공하면 됩니다.
-   테마(또는 자체 `DESIGN.md` 파일)를 선택/가져올 수 있습니다.
-   한 번의 클릭으로 해당 프론트엔드 페이지를 생성합니다.

곧 오픈소스로 공개될 예정이니 기대해 주세요.

## NotebookLM CLI

여기서 제가 직접 만든 NotebookLM CLI 프로젝트도 추천하고 싶습니다. 에이전트가 NotebookLM 내에서 작업을 자동화할 수 있게 해줍니다.

저는 이 NotebookLM CLI와 Stitch SDK를 사용하여 나만의 일일 팟캐스트 앱과 웹사이트를 구축했습니다.

웹사이트의 원리는 다음과 같습니다:
-   NotebookLM이 리서치 + 팟캐스트를 생성합니다.
-   Whisper를 사용하여 자동 자막 생성 및 푸시합니다.
-   GitHub를 통해 Vercel에 업로드합니다.
-   오디오를 Cloudflare R2에 자동으로 푸시하고 URL로 변환합니다.
-   URL을 페이지에 삽입하여 완전 자동화된 전달을 구현합니다.

왜 이렇게 번거롭게 할까요? 오디오 파일이 크고, R2의 무료 티어가 많은 번거로움을 덜어주기 때문입니다.

이제 제 작업 흐름에 통합되었습니다. 주제만 제공하면 나머지는 기본적으로 NotebookLM에 맡겨져 토큰을 절약하면서도 좋은 결과를 얻을 수 있습니다.
