## Claude Code를 거의 1년 동안 사용하다가, 누군가가 가장 알아두면 좋은 팁 45가지를 모두 정리해 놓은 것을 발견했습니다.

↓ 결론부터 말하자면: 대부분의 사람들은 잠재력의 약 30%만 사용하고 있습니다.

───

🔋 **토큰 절약 / 비용 절감**

Claude Code의 기본 시스템 프롬프트는 약 2만 토큰을 소모합니다. 작업을 시작하기도 전에 컨텍스트의 10%를 태워버리는 셈이죠. 패치 후에는 약 9천 토큰으로 줄일 수 있어, 대화당 약 1만 토큰을 절약할 수 있습니다. 장기간 사용하면 차이가 상당합니다.

MCP 도구의 경우 지연 로딩(`ENABLE_TOOL_SEARCH: "true"`)을 활성화하는 것이 좋습니다. 도구가 필요할 때만 로드되어 불필요하게 공간을 차지하지 않습니다.

`/compact`를 적극적으로 사용하거나 `HANDOFF.md`를 작성하여 새 세션으로 전환하세요. 자동 압축을 기다리는 것보다 훨씬 정확합니다.

───

🧠 **컨텍스트 관리**

AI 컨텍스트는 우유와 같아서, 신선할수록 좋습니다. 새 주제는 새 세션에서 시작하고, 억지로 이어나가지 마세요.

매우 긴 대화의 경우 요약에 의존하지 마세요. "반쪽 복제"를 사용하세요: 대화의 후반부에 있는 실제 메시지만 유지하는 것입니다. 이렇게 하면 정보가 왜곡되지 않고 보존됩니다. 컨텍스트가 85%를 초과하면 자동으로 트리거되는 훅을 설정할 수 있습니다.

───

🚀 **워크플로우 속도 향상**

음성은 타이핑보다 훨씬 빠릅니다. 로컬 Whisper 모델이 가끔 전사 오류를 일으켜도 Claude가 이해할 수 있습니다. 비행기 안에서도 사용할 수 있습니다.

멀티태스킹을 위해서는 "캐스케이드 방식"을 권장합니다: 터미널 탭 3~4개를 나란히 열고 왼쪽에서 오른쪽으로 순환하며 작업을 처리하세요. 이렇게 하면 여기저기 뛰어다니지 않고 한 방향에 집중할 수 있습니다.

Git worktree: 여러 브랜치를 별도의 디렉토리에서 독립적으로 작업할 수 있어 간섭이 없습니다. 여러 탭과 결합하면 효율성이 두 배로 높아집니다.

Claude가 Reddit 같은 사이트에 접근할 수 없나요? Gemini CLI를 대체 도구로 구성하고 tmux를 통해 중계하면 연구 작업도 계속할 수 있습니다.

───

🔍 **코드 품질**

TDD는 Claude Code와 놀랍도록 잘 맞습니다: 먼저 실패하는 테스트를 작성하고 커밋한 다음, Claude가 테스트를 통과하는 코드를 작성하게 하세요. 명확한 목표가 있으면 정확도가 훨씬 높아집니다.

GitHub Actions 로그를 직접 뒤질 필요가 없습니다. `/gha <url>`로 한 번에 분석하세요: 간헐적인 오류인지, 문제를 일으키는 커밋인지, 아니면 설정 문제인지 자동으로 판단하여 정확한 진단을 제공합니다.

정기적으로 `cc-safe`를 실행하여 `settings.json`을 스캔하고, `rm -rf`나 `sudo` 같은 위험한 명령어를 실수로 승인하지 않았는지 확인하세요.

───

🤖 **고급: 멀티 에이전트 오케스트레이션**

Claude Code를 Docker 컨테이너에 넣고 `--dangerously-skip-permissions`로 시작하여 자율적으로 반복, 수정, 테스트하게 하세요. 문제가 발생해도 샌드박스 내에서 격리되어 호스트를 안전하게 보호합니다.

더 나아가: 외부 Claude가 tmux를 통해 컨테이너 내부의 작업자 Claude를 제어하게 하세요. 명령어를 보내고, 출력을 읽고, 결과를 가져오는 모든 과정을 터미널을 수동으로 전환할 필요 없이 처리합니다. 완전 자동화된 에이전트 파이프라인입니다.

대규모 코드베이스를 병렬로 분석해야 하나요? Claude가 여러 하위 에이전트를 동시에 실행하여 작업을 여러 모듈에 분산시키세요. 작업 복잡도에 따라 컴퓨팅 파워를 할당하여 Opus / Sonnet / Haiku 모델을 지정할 수도 있습니다.

───

📁 **설정 습관**

`CLAUDE.md`는 빈 파일에서 시작하세요. 반복해서 말하게 되는 내용만 추가하세요. 간결하게 유지하세요. 정기적으로 검토하고 오래된 내용은 삭제하세요.

터미널 별칭을 설정하세요: `c = claude`, `ch = claude --chrome`. 매일 절약되는 시간이 적지 않습니다.

자동 업데이트를 비활성화(`DISABLE_AUTOUPDATER: "1"`)하여 새 버전이 시스템 프롬프트 패치를 망가뜨리는 것을 방지하세요.
