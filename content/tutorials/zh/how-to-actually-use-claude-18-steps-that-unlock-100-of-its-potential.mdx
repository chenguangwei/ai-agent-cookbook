# 如何真正使用 Claude：解锁其 100% 潜能的 18 个步骤

Claude 已经问世两年了。大多数每天使用它的人，仍然只发挥了其 10% 的能力。

不是因为它复杂，而是因为没人向他们展示过另外 90% 是什么样子。

本指南将解决这个问题。读完它，你将能够以记住你、理解你、并按照你实际思考方式工作的方式来设置 Claude。你还将学会如何用它来做大多数人从未尝试过的事情。

## 从这里开始

**1 - 创建一个项目，而不是一个聊天**

每次你打开一个新的 Claude 聊天，它都是从零记忆开始的。它不知道你的名字、你的工作、你的目标，或者你喜欢如何沟通。你会在前几条消息中重新解释自己，或者你不解释，然后 Claude 给你一些通用的、不符合你实际工作方式的内容。

项目解决了这个问题。一个项目是一个持久的工作空间，Claude 会在其中的每个对话中保持上下文。你只需设置一次，之后每次会话开始时，Claude 就已经知道你是谁了。

前往 Claude，点击侧边栏中的“项目”，然后创建一个新项目。根据你的使用计划，将其命名为“工作”或“个人”之类的名称。接下来的所有步骤都在这个项目内进行。

**2 - 告诉 Claude 你是谁**

在 Claude 能很好地帮助你之前，它需要了解你。大多数人完全跳过这一步，然后奇怪为什么 Claude 给他们的答案感觉有点不对劲。

将以下内容粘贴到你的项目中，并如实填写每个字段。你越具体，每个回复就会变得越好。

```text
我的名字是 [你的名字]。

我的工作是 [你的角色或职业]。我的主要职责是 [你日常做的 2-3 件事]。

目前我最大的目标是 [你正在努力的 1-3 个具体目标]。

我主要使用 Claude 来做 [列出你的主要用例——写作、研究、分析、学习、编码等]。

我的背景和知识水平：[你擅长什么、正在学什么、对什么不熟悉]。

我喜欢接收信息的方式：[例如：直接简洁 / 详细并附有例子 / 分步指导 / 不要项目符号 / 短段落]。

我不想要的东西：[例如：不要添加免责声明、不要使用企业语言、不要重复我刚说过的话、不要以“好问题”开头]。

我关心的话题和领域：[你的兴趣、行业、细分领域]。
```

将此内容保存到你的项目知识库中。Claude 会在该项目中的每次对话开始时读取它。

**3 - 将其转化为自定义指令**

粘贴你的背景信息是一个好的开始。但自定义指令更进一步。它们不仅告诉 Claude 你是谁，还确切地告诉它默认情况下应该如何与你互动。

在你填写完上面的模板后，将以下提示词粘贴给 Claude：

```text
根据你告诉我关于我的一切，为这个 Claude 项目写一套自定义指令。

指令应该：
- 描述我是谁以及我做什么
- 设定我的默认沟通风格和格式
- 告诉 Claude 在与我合作时永远不要做什么
- 定义我期望每次回复的语气
- 包含我希望在每个会话中拥有的任何默认行为

用第二人称写，就像 Claude 在阅读关于如何帮助我的规则一样。要具体。不要有泛泛的建议。不超过 400 字。
```

将输出结果粘贴到你的项目指令中。这将成为 Claude 在该项目内每个对话的永久操作模式。

## Claude 并非你所想的那样

**4 - Claude 不是搜索引擎**

大多数人使用 Claude 的方式就像使用 Google 一样。他们输入一个问题，然后等待一个答案。这是使用它的最低价值方式。

Claude 不是一个检索工具。它是一个思考伙伴。它不仅仅是提取信息，它还会推理、综合、辩论，并基于上下文进行构建。一旦你把它当作搜索引擎，它的效用就降低了 80%。

停止问 Claude“某物是什么”。开始让 Claude 帮助你思考某件事。

不要这样问：“什么是提示缓存？”

试试这样问：“我正在构建一个工作流，每次会话会调用 Claude 20 次。请向我解释提示缓存是如何工作的，以及考虑到那个上下文，它是否真的能降低我的成本。”

第二个提示词给 Claude 提供了一个与你一起解决的问题。第一个提示词只是让它背诵一个定义。

**5 - 先让 Claude 问你问题**

这是几乎没人使用的最强大的技巧之一。在 Claude 开始任何复杂任务之前，告诉它先向你收集信息。

当 Claude 在开始之前问你问题时，输出结果会好得多，因为它建立在正确的基础上。没有这一步，Claude 会做出假设，而你会花时间纠正那些本可以一次就做对的事情。

在任何重要任务之前使用这个技巧：

```text
在开始之前，先问我 5 个最重要的问题，这些问题能帮助你做好这件事。

我回答之后，你再开始。
```

或者针对特定任务：

```text
我需要你帮我写一封给潜在客户的冷邮件。

在写任何东西之前，先问我你需要知道什么，才能让这封邮件真正出色，而不是泛泛而谈。
```

## 即使是普通用户也不知道的技巧


### 6 - 风格克隆

当 Claude 在没有示例的情况下模仿你的语气写作时，它实际上是在用自己的语气写。输出语法正确，但语气完全不对。听起来像 AI，因为它本来就是。

给 Claude 三到五个你自己写的样本。让它分析你的模式，而不仅仅是描述你的风格。经过分析后，它写出来的东西就像你写的，而不是像一个经过润色的企业助手。

```text
以下是我写作的 3 个示例：

[粘贴示例 1]
[粘贴示例 2]
[粘贴示例 3]

详细分析我的写作风格。关注：句子长度、节奏、词汇选择、段落开头和结尾的方式、我避免使用的表达、正式或非正式的程度，以及任何使我的写作与众不同的模式。

在此之后，当我要求你为我写任何东西时，请完全匹配这种风格。不要默认使用你自己的模式。
```

### 7 - 把 Claude 当作你的陪练伙伴

大多数人让 Claude 帮他们完善想法。这意味着 Claude 会基于你所说的内容进行补充、扩展。你得到的是附和和阐述。

这在某些时候很有用。但这并不是检验一个想法是否可靠的方式。

在敲定任何计划、决策或文章之前，让 Claude 去攻击它。不是批评，是攻击。这两者有本质区别。

```text
这是我的计划：[描述你的计划]

你的任务是摧毁它。找出我可能犯错的每一个假设。找出所有可能导致失败的方式。尽可能有力地反驳我的立场。不要客气。不要添加任何保留意见。只管攻击。

之后，请为我的立场做最有力的辩护。构建一个最强大的理由来说明为什么我是对的。

然后，告诉我你真实的想法。
```

### 8 - 扩展思考

大多数 Claude 用户从未开启过这个功能。扩展思考是一种模式，Claude 会在给出答案之前逐步推理问题，而不是直接输出结果。

对于简单任务，你不需要它。但对于复杂决策、分析，或者任何你希望 Claude 真正思考而不是模式匹配的问题，请开启它。

在 Claude 中，发送消息前点击大脑图标。或者在提示词中添加以下内容：

```text
在回答之前，请仔细思考这个问题。

逐步解决问题，展示你的推理过程，指出你不确定的地方，然后给出你的结论。
```

在难题上，输出质量的差异非常显著。

### 9 - Claude 为 Claude 写提示词

这是你可以做的最被低估的事情之一。如果你不确定如何为特定任务向 Claude 提问，就让 Claude 为你写提示词。

Claude 知道什么样的指令能产生更好的结果。让它为你利用这些知识。

```text
我需要 Claude 帮我 [描述你的实际任务]。

请为这个任务写出最好的提示词。

包括角色、上下文、格式要求，以及任何能改善输出的约束条件。

然后立即使用这个提示词。
```

## 如何用更少的 Token 获得更多


### 10 - 指定输出长度

Claude 的默认行为是尽可能多地写它认为合适的内容。这通常比你需要的要多，这意味着会消耗更多 Token，花费更多阅读时间，并在输出中产生更多噪音。

在 Claude 开始之前，明确告诉它你想要的答案长度。

```text
最多用 3 句话回答。
```

```text
给我 5 个要点。不要解释。只要要点。
```

```text
用 150 字以内写完。
```

这一个指令就能在大多数任务上减少 40% 到 60% 的 Token 消耗，而不会损失你实际需要的任何价值。

### 11 - 去掉开场白

Claude 的每个回复默认都会以一些你没要求的内容开头。“好问题。让我为你详细解释。”或者完整复述你刚刚说过的话。或者免责声明。或者结尾总结，重复它刚刚告诉你的所有内容。

你并没有要求这些。它们消耗 Token，也浪费你的时间。

在你的自定义指令中添加以下内容：

```text
永远不要以开场白、肯定性话语或复述我的问题来开始回复。

直接给出答案。

除非我特别要求，否则不要在结尾添加总结。

除非话题确实需要，否则不要添加免责声明。
```

### 12 - 不要每次对话都重新解释自己

如果你在每个新对话中都粘贴相同的背景信息，那么你每次都在浪费 Token，并且正在养成一个随着 Claude 使用量增加而让你付出更多代价的习惯。

这正是项目和自定义指令的用途。把你的上下文放进去一次。让 Claude 在每个会话开始时自动读取。再也不需要粘贴你的背景信息了。

如果你还没有使用项目功能，在阅读本文其他内容之前，请先从这里开始。

### 13 - 新话题开启新对话

Claude 会携带对话中之前说过的所有上下文。当你在一个长对话中切换话题时，Claude 仍然加载着之前的所有上下文。这意味着每次回复都会消耗更多 Token，处理速度变慢，并且之前对话中的上下文会影响到你的新话题。

当你切换到不相关的话题时，在你的项目中开启一个新的对话。你保留了项目的记忆。你丢掉了不相关的包袱。

## 开箱即用


以下是可直接复制使用的完整提示词。

**14 - 用类比理解任何事物（费曼学习法）**

克劳德默认给出的解释通常技术上准确但实际用处不大。它们使用与你要理解的事物相同的词汇，结果你只得到了定义，却没有真正理解。

费曼学习法通过简化来强制理解。如果克劳德不能用类比和通俗语言解释清楚，说明解释还不够清晰。这个提示词适用于从投资到量子物理，再到特定 API 工作原理的任何内容。

```text
请仅用类比和日常例子向我解释[主题]。不要使用专业术语。假设我对这个领域毫无背景知识。

每给出一个类比后，通过问我一个问题来检查我是否真正理解了。根据我的回答，要么深入讲解，要么调整解释方式。

一直进行下去，直到我能用自己的话、不使用任何技术术语向你复述出来为止。
```

**15 - 根据你的实际旅行方式定制行程**

大多数旅行规划从目的地开始，以你在任何旅游博客上都能找到的通用行程结束。克劳德可以做得不同：围绕你特定的旅行风格、节奏、预算和真正重要的事情来制定计划，而不是那些必看清单上的内容。

关键在于提供关于你的真实信息，而不仅仅是日期和地点。

```text
我正在计划去[目的地]旅行。我将在那里待[天数]。我的预算大约是每天[金额]，包括住宿。

以下是我实际的旅行方式：[描述你的风格——你喜欢悠闲的早晨还是紧凑的行程，旅游景点还是本地小众地点，博物馆还是美食，活跃还是放松，独自一人还是结伴同行等]。

我想避免的事情：[人群拥挤、旅游陷阱、昂贵的餐厅、长时间的交通等]。

请为我制定一个符合这些要求的每日行程。每天都要包括住宿地点、活动安排、用餐地点以及需要了解的任何后勤信息。标注任何需要提前预订的事项。
```

**16 - 月度支出分析及实际结论**

大多数人查看银行账单时，只会模糊地对自己的消费感到不安，却不了解实际情况。克劳德可以将原始数据转化为清晰的画面，显示你的钱花在了哪里，以及你应该采取什么措施。

粘贴真实数据（而非估算）时效果最佳。

```text
我将粘贴上个月的支出。请分析并告诉我：

1. 我在哪些类别上花费最多
2. 与我描述的目标相比，哪些支出看起来异常
3. 哪些支出可以削减，且不会显著影响我的生活质量
4. 我可能在哪些重要方面支出不足
5. 一个能产生最大财务影响的具体改变

以下是我的支出：[粘贴你的银行账单或支出清单]

我目前的财务目标：[描述你正在努力实现的目标——存更多钱、还清债务、投资等]
```

**17 - 克劳德作为你的个人思考伙伴**

大多数人的生活中没有这样一个角色：不带评判地倾听，提出恰当的问题，帮助你理清困扰的事情，而不强加自己的观点。克劳德可以扮演这个角色，但前提是你给它正确的指令。

这不是心理治疗。这是一种结构化的自我反思，借助外部视角帮助你更清晰地思考。

```text
我想讨论一下我正在处理的事情。你的任务不是立即给我建议。

首先，通过提问来全面了解情况：实际发生了什么，我的感受如何，我已经尝试过什么，以及我希望达到什么结果。

在你完全了解情况后，复述你听到的内容——不仅是事实，还有表面之下的深层问题。

然后，也只有到那时，才给出你的观点。要诚实，而不是安慰。告诉我你真实的想法，包括任何我可能不想听到的内容。

以下是我在想的事情：[描述你想理清的问题]
```

**18 - 在投入之前对任何商业想法进行压力测试**

大多数商业想法之所以失败，是因为人们在测试之前就爱上了它们。他们花了数月时间构建没人想要的东西，因为他们从未诚实地问过这个想法是否真的可行。

克劳德可以充当一个无情的初步过滤器。不是为了扼杀想法，而是在它们耗费你的时间和金钱之前找到真正的问题。

```text
我有一个商业想法，想在投入大量时间之前对其进行压力测试。

以下是这个想法：[详细描述——它是什么，目标用户是谁，如何盈利，为什么你认为它可行]

你的任务是找出它所有的问题。具体来说：

1. 我可能犯了哪些错误假设
2. 谁已经在做类似的事情，我为什么会输给他们
3. 目标客户为什么可能不会为此付费
4. 这个想法要成功需要满足什么条件，这些条件的可能性有多大
5. 这个想法最大的问题是什么

要具体。像"市场可能还没准备好"这样的泛泛风险没有用。请给出每个问题的真实版本。

之后，告诉我这个想法需要变成什么样才能真正可行。
```



## 真正的要点


克劳德并不比你更聪明。它没有比你更好的想法。它拥有的是无限的耐心、广博的知识，以及从你未曾考虑过的角度思考问题的能力。

从克劳德那里获得最大收益的人，不是那些提问最好的人。而是那些让它了解自己、提供真实背景信息、知道如何将其作为伙伴而非工具来使用的人。

大多数人会阅读本文，然后继续以他们一贯的方式打开克劳德。

一次性设置好。永久改变你的工作方式。
